ChatGPT Plugins Store正式发布，类App Store和微信生态加速AI革命

AI 人工智能技术革命前夜，你准备好了吗？如果说ChatGPT-3的发布是人工智能技术革命迈出的第一步，那GPT-4发布就是正式叩响了开启人工智能技术革命的大门了，这次插件的发布，将会大大加快在各个行业和领域的革命速度。

Plugins 插件功能发布

今天 ChatGPT 对外宣布了流传已久的 Plugins 功能 简单来说就是允许第三方为 ChatGPT 开发插件，将自己的服务集成到 ChatGPT 的对话窗口中。

从演示来看，插件已经包括了酒店航班预订、在线购物、法律知识库、文字生成语音还有大家最爱的 Stephen Wolfram，你可以指定他来回答最专业的数理知识，不用再听 ChatGPT 胡编了。。

想象空间是不是一下打开？ ChatGPT 就是智能时代的操作系统，大语言模型就是智能内核 - Connect your App to LLMverse 的时代已来

openai 提供了一个Browsing 网页浏览插件和一个 Code interpreter 代码执行插件，并开源了一个 Retrieval 检索插件的代码。 插件仍然处于试验阶段，但我们认为在这个方向上有一些很棒的东西；这是一个非常需要的功能。

Browsing 官方插件

Browsing 插件将用互联网上最新的信息来回答你的问题，给出它的搜索步骤和内容来源链接。其实 OpenAI 的 WebGPT 还有 Google 的 LaMDA2 都做过类似尝试，通过自主的实时检索，回答超越训练语料库范围的内容，只不过这一次是通过插件形式集成到了 ChatGPT。Browsing 使用了 Bing API，真是好兄弟相互帮助 。。

Code Interpreter 代码解释器插

Code Interpreter 是一个实验性的模块，它创造了一个可以运行 Python 解释器的沙盒，在防火墙控制下有短暂的存储空间，能处理文件上传和下载，整个虚拟空间在会话期有效。在最初的用户研究中，已经确定代码解释器可以做： · 解决数学问题，包括定量和定性 · 进行数据分析和可视化 · 在不同格式之间转换文件 一个会写代码的语言模型加上能执行代码的解释器，那以后是不是可以自己升级自己了

Retrieval 用于开发 ChatGPT 插件的开源框架

Retrieval 用于开发 ChatGPT 插件的开源框架，它让 ChatGPT 可以访问个人与组织内部经许可的数据，例如文件、笔记、电子邮件或公共文件等，它允许用户通过提问或用自然语言表达需求。Retrieval 提供了一个自托管的方案，开发者可以部署自己的插件，并在ChatGPT注册。利用 OpenAI Embedding，并允许选择一个矢量数据库（Milvus、Pinecone、Qdrant etc）来索引和搜索文档，信息源可以通过 webhooks 与数据库进行同步。 Retrieval plugin repository https://github.com/openai/chatgpt-retrieval-plugin

第三方开发的 Wolfram 插件

最后，分享一篇来自 Wolfram 的文章，ChatGPT现在可以通过 Wolfram 插件调用 Wolfram｜Alpha 的知识库，实现了静态知识结合了 LLMs 之后的“计算超能力”。这一切还处于早期阶段，但表现已经令人印象深刻，甚至可能是革命性的！

请注意，此功能目前仅适用于某些 ChatGPT Plus 用户；有关详细信息，请参阅 OpenAI 的公告。

仅仅两个半月过去了… 早在一月份，我就写过关于将ChatGPT连接到Wolfram|Alpha的可能性。而今天，仅仅两个半月过去，我很高兴地宣布这已经实现了！得益于我们的团队和OpenAI的英雄般的软件工程，ChatGPT现在可以调用Wolfram|Alpha和Wolfram语言，给它提供我们可以称之为“计算超能力”的东西。虽然这仍然处于非常早期的阶段，但已经非常令人印象深刻，我们可以开始看到“ChatGPT + Wolfram”这个组合有多么强大（甚至可能是革命性的）。

在一月份，我提出了这样一个观点，即作为一个LLM神经网络，ChatGPT虽然在文本生成方面表现出了非凡的能力，例如从网络等读取资料后生成“类似”资料，但不能指望它自己完成实际的复杂计算，或者系统地产生正确的（而不仅仅是“看起来大致正确的”）数据等。但当它连接到Wolfram插件时，它可以完成这些事情。因此，这是我（非常简单的）一月份的第一个示例，现在由安装了“Wolfram超级能力”的ChatGPT完成：

开发者生态观察

OpenAI开放插件的目的是绑定开发者生态，把自己打造成互联网新的操作系统和流量入口。 相比于过去半年仅提供API帮助建设不同产品，所有的产品都相当于在为ChatGPT提供功能和现实世界接口。一越从幕后转到台前。 联网数据，运行代码，开放交互是这次OpenAI生态策略的三大主题，每一条都比你想象得要更有长远影响。

  1. 过去的ChatGPT在遇到一些问题时会提醒你它的训练数据并不包括这一内容，但是在添加Browsing插件后就可以自主判断什么时间去互联网获得更多信息以及如何获取筛选和解读这些数据。将ChatGPT联网意味着或许会更进一步改变我们获取信息的习惯进一步提高效率，但是也意味着多重风险 - 模型被有意投喂大量恶意数据，通过AI作恶也会更加容易。
  2. 旧版ChatGPT其实是可以通过指令使其“模拟”成一个命令行界面运行代码，但是结果其实是内在逻辑分析后再伪装成代码结果。使用Code Interpreter插件后就可以在对话中直接进行Python代码运作。这不是仅仅允许使用者运行python代码那么简单，而是意味着一些复杂问题ChatGPT会自己撰写代码，运行后辅助分析。AI真正会使用代码就像当初动物学会使用工具一样。
  3. 能够通过ChatGPT对话直接通过Instacart上面买菜、通过Kayak预定旅行交通和住宿，通过Klarna买到各个平台比价最便宜的商品只是冰山一角。通过链接Zapier意味着我们可以更容易通过ChatGPT定制我们的工作流，自动回复邮件、撰写新闻稿、进行代码Review都不需要人工介入。几乎你在互联网上能做的一切事情ChatGPT都可以去替你做。

这一点其实刚好契合了我昨天所分享的AI公司护城河的思考，Retrieval这一个案例插件直接干掉大多数summarization企业。希望每一位buidler共勉。大势不可挡，但是漩涡里也能涌现更多弄潮儿。

总结

有没有人经历过苹果app store开发者生态的从0到1的爆发期…还有微信开发者生态生态从0到1的爆发期，个人观点ChatGPT必定会活，并定也会发生从0到1的爆发期，我错过了苹果的和微信的两次，想想当时自己还是太年轻了，希望这次自己不要错过。

苹果是西方美国的，进入中国才能带动整个移动产业生态，特斯拉也是西方美国的2020年进入中国带动新能源汽车产业的爆发。虽然前两者都是基于硬件，跟制造业有很大的强关系，与openAI的ChatGPT不同是人工智能，目前来说，还是属于纯软件的生态。个人观点只要国内这方面的大厂不争气搞不出来自然语言大模型，在这样的背景下，肯定不会完全feng死。即使feng死，想想搜索baidu和google，有人为什么不想用B而就是想法设法要用G吧！这种来自政策的风险分析评估，有时间再和大家分享。

总之，如果要想赶上这次浪潮，也成为站在风口上的那头猪，现在进入还算是变革的前夜，早期有很多的红利，想想易懂互联网早期的红利窗口期吧，到现在也不过10年左右，这次如果错过了就真的不知道下一次的技术革命周期会要再等10年、20年、30年。。。？
